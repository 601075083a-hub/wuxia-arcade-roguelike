# 武侠街机 Roguelike 产品规格与 Demo 计划

## 1. 产品定位

一句话定位：一款以武侠为题材、以 Deckbuilder 为构建骨架、以街机搓招为核心出招方式的轻量 Roguelike 游戏。

核心差异点不是“武侠卡牌”，而是“把牌型判断替换成搓招输入”。玩家每回合抽到的是方向和攻击碎片，通过排列碎片组成招式，再用外功和内功改变可用配方、伤害乘区和战斗规则。

## 2. Demo 目标

本 Demo 的目标是验证完整闭环是否成立，而不是一次性做完所有系统。

首版要验证的问题：

- 玩家是否能在 3 分钟内理解“抽碎片 -> 排列碎片 -> 搓出招式 -> 造成效果”。
- 搓招是否比普通出牌更有爽感和记忆点。
- 外功作为“配方解锁”，内功作为“规则改写”，是否能形成清晰的构建方向。
- 一局 10-15 分钟的流程是否适合微信小游戏的碎片化场景。

首版不追求：

- 完整商业化。
- 大量角色。
- 大规模天赋树。
- 长线数值养成。
- 复杂剧情和大型资源内容。

## 3. 核心循环

Demo 的核心循环：

```text
选择角色
  -> 进入跑图
  -> 战斗
  -> 获得奖励
  -> 调整牌组 / 获得外功 / 获得内功
  -> 选择下一个节点
  -> 击败 Boss
  -> 结算
```

局内循环重点：

- 跑图提供选择压力。
- 战斗验证搓招手感。
- 奖励改变构建方向。
- Boss 检验构建成果。

## 4. Demo 范围

采用“完整但窄”的范围。

首版必须包含：

- 1 个主角。
- 6 种基础碎片：上、下、左、右、拳、腿。
- 10-12 个基础或外功招式。
- 6-8 个外功。
- 6-8 个内功。
- 5 个普通或精英敌人。
- 1 个 Boss。
- 1 套基础跑图流程。
- 商店、事件、奖励、结算的最小可用版本。

首版暂缓：

- 多角色体系。
- 完整 60 点天赋树。
- 多章节剧情。
- 多 Boss 池。
- 战令、皮肤、付费角色、广告接入。

## 5. 战斗规则

### 5.1 基础碎片

战斗中的牌不是传统技能牌，而是招式碎片。

基础碎片共 6 种：

| 类型 | 碎片 | 用途 |
| --- | --- | --- |
| 方向 | 上 | 组成方向序列 |
| 方向 | 下 | 组成方向序列 |
| 方向 | 左 | 组成方向序列 |
| 方向 | 右 | 组成方向序列 |
| 攻击 | 拳 | 触发拳系招式 |
| 攻击 | 腿 | 触发腿系招式 |

### 5.2 回合流程

每场战斗按回合推进：

1. 抽牌阶段：从牌库抽取 5 张碎片。
2. 搓招阶段：玩家最多出招 3 次。
3. 结算阶段：结算招式伤害、护甲、状态和敌人行动。
4. 弃牌阶段：未保留的手牌进入弃牌堆。
5. 洗牌阶段：牌库为空时，将弃牌堆洗回牌库。

建议初始参数：

| 参数 | 首版值 |
| --- | --- |
| 初始牌组 | 12 张 |
| 每回合抽牌 | 5 张 |
| 手牌上限 | 7 张 |
| 每回合出招次数 | 3 次 |
| 搓招长度上限 | 5 个碎片 |

### 5.3 搓招规则

玩家从手牌中选择碎片，并按顺序排列。

若排列结果命中已解锁配方，则触发对应招式；若未命中，则不能出招，或只产生低效的普通攻击。首版建议采用“未命中不能出招”，降低规则歧义。

示例配方：

| 输入 | 招式 | 效果定位 |
| --- | --- | --- |
| 下 + 右 + 拳 | 顺风拳 | 单体中伤害 |
| 右 + 右 + 拳 | 连环拳 | 低伤害连击 |
| 下 + 上 + 腿 | 飞踢 | 高伤害击退 |
| 左 + 下 + 右 + 拳 | 伏虎拳 | 重击破防 |
| 下 + 右 + 上 + 腿 | 旋风腿 | 群体伤害 |

首版配方设计原则：

- 2-3 段招式用于新手期。
- 4-5 段招式用于构建成型后的高收益。
- 每个外功至少提供 1 个容易触发的招式和 1 个高收益招式。

## 6. 外功系统

外功定位：招式配方表 + 类型增幅。

玩家获得外功后，可以解锁新的搓招配方，并获得某类招式的增强效果。

首版装备结构：

| 位置 | 数量 | 作用 |
| --- | --- | --- |
| 主外功 | 1 | 决定核心配方和主要构建方向 |
| 副外功 | 1 | 补充辅助配方 |
| 招式强化 | 1-2 | 增强拳系、腿系、破防、连击等标签 |

首版外功候选：

| 外功 | 构建方向 | 示例效果 |
| --- | --- | --- |
| 金刚伏魔拳 | 拳系破防 | 解锁重拳配方，拳系伤害提升 |
| 太极拳 | 方向转换 | 左方向可视为任意方向一次 |
| 降龙掌 | 高爆发 | 每回合第一次拳系招式增伤 |
| 暴雨梨花 | 群体中毒 | 腿系或暗器型招式附带中毒 |
| 独孤九剑 | 连段爆发 | 连续命中不同配方后增伤 |
| 罗汉拳 | 稳定防守 | 出拳后获得护甲 |

外功的首版目标是让玩家看见明确构建路线，而不是堆数值。

## 7. 内功系统

内功定位：规则改写。

内功不只是增加伤害，而是改变抽牌、弃牌、碎片转换、出招次数或容错机制。

首版装备结构：

| 位置 | 数量 | 作用 |
| --- | --- | --- |
| 主内功 | 1 | 决定核心规则变化 |
| 辅内功 | 1 | 提供辅助变化 |

首版内功候选：

| 内功 | 构建方向 | 示例效果 |
| --- | --- | --- |
| 九阳神功 | 速攻 | 每回合第一次出招不消耗出招次数 |
| 北冥神功 | 弃牌转化 | 每弃 1 张碎片，下回合额外抽牌 |
| 易筋经 | 碎片转换 | 上下、左右可互换一次 |
| 小无相功 | 攻击转换 | 拳和腿可互换一次 |
| 先天功 | 精简牌库 | 战斗开始移除临时弱碎片 |
| 龙象般若功 | 蓄牌爆发 | 手牌越多，招式伤害越高 |

内功设计原则：

- 每个内功都要改变玩家决策。
- 避免只做“伤害 +X%”。
- 优先支持和外功产生组合效果。

## 8. 关卡结构

Demo 采用 3 层跑图。

```text
第 1 层：初入江湖
  普通战斗 -> 普通战斗 / 事件 -> 精英战斗

第 2 层：江湖纷争
  普通战斗 -> 商店 / 事件 -> 普通战斗 -> 精英战斗

第 3 层：决战之巅
  普通战斗 -> 休息 / 商店 -> Boss
```

节点类型：

| 节点 | 首版作用 |
| --- | --- |
| 普通战斗 | 提供基础奖励和构建推进 |
| 精英战斗 | 掉落更强外功或内功 |
| 商店 | 购买碎片、删除碎片、购买外功 |
| 事件 | 提供风险收益选择 |
| 休息 | 回复生命或强化一次招式 |
| Boss | 检验本局构建 |

首版敌人目标：

- 第 1 层敌人只教会基础攻击、防御和意图。
- 第 2 层敌人加入护甲、状态、被动。
- 第 3 层 Boss 加入搓招反击或阶段变化。

## 9. 局外系统

首版只做轻量局外系统，避免过早把开发量投入长线养成。

推荐方案：

- 只做 1 个主角。
- 主角拥有 1 个初始被动。
- 通关或失败后只展示结算数据。
- 天赋树先做占位入口，或只做 3-5 个轻量升级项。

暂不实现完整三路线 60 点天赋树。

局外系统的首版目的：

- 给玩家轻微成长感。
- 支撑 Demo 闭环。
- 不抢占搓招战斗验证的开发资源。

## 10. UI 页面清单

首版需要以下页面：

| 页面 | 主要内容 |
| --- | --- |
| 主界面 | 开始游戏、继续游戏、设置入口 |
| 角色选择 | 当前主角、初始牌组、初始被动 |
| 跑图页面 | 当前层数、节点选择、路线预览 |
| 战斗页面 | 手牌碎片、搓招区、敌人、生命、护甲、意图 |
| 奖励页面 | 选择碎片、外功、内功或金币 |
| 商店页面 | 购买、删除、刷新 |
| 事件页面 | 事件描述、选项、结果 |
| 休息页面 | 回复或强化 |
| 结算页面 | 胜利/失败、用时、击败敌人、构建摘要 |

战斗页面是最高优先级页面。

战斗 UI 必须清楚表达：

- 当前手牌有哪些碎片。
- 玩家已经排列了哪些碎片。
- 当前排列是否能形成招式。
- 招式会造成什么效果。
- 敌人下一步要做什么。

## 11. 微信小游戏设计约束

首版按微信小游戏方向预留约束，但不在 Demo 中接入商业化。

设计约束：

- 单局目标时长 10-15 分钟。
- 单场普通战斗 1-2 分钟。
- 单次节点选择不超过 20 秒。
- 触屏操作优先：点击碎片、拖拽排序、滑动方向都可作为后续方案。
- 美术资源优先轻量化，适配主包和分包限制。
- UI 字号和按钮尺寸要适合手机竖屏。

首版商业化只作为后续规划，不进入实现范围。

## 12. 验收标准

Demo 完成后，应满足以下标准：

- 玩家能完成从开局到 Boss 结算的一整局。
- 玩家能理解基础碎片、搓招、外功、内功的关系。
- 玩家能在 10-15 分钟内完成一局。
- 玩家能在战斗中主动追求更复杂的搓招。
- 至少存在 3 种可感知构建方向，例如拳系爆发、腿系群攻、蓄牌爆发。
- 外功和内功不是纯数值堆叠，而是能改变出招选择。
- 失败后玩家能明确知道下一局可以如何调整构建。

## 13. 后续扩展

Demo 验证通过后，再考虑扩展：

- 多角色。
- 完整天赋树。
- 更多门派外功。
- 更多内功组合。
- 每日挑战。
- 成就系统。
- 皮肤和特效。
- 激励视频广告。
- 战令或赛季内容。

商业化优先级低于核心玩法验证。

## 14. 当前结论

该方向值得继续推进，但执行上必须先验证搓招战斗是否成立。

首版应做成“完整但窄”的可玩 Demo：系统闭环完整、内容数量克制、战斗体验优先。只要搓招构建的乐趣成立，武侠门派、外功内功、局外养成都有足够扩展空间。
